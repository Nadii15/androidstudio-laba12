# 🎓 Grade Clicker App (Кликер Оценок)

**Краткое описание:**  
Интерактивное приложение-кликер для набора баллов по курсу «Разработка мобильных приложений». Пользователь кликает по изображению, набирая баллы (+5 за клик). При достижении определённых порогов (50, 70, 90, 100 баллов) изменяются: отображаемая оценка, фоновое изображение и цвет интерфейса. Приложение демонстрирует работу жизненного цикла Activity (Activity Lifecycle) и сохранение состояния при изменении конфигурации устройства.

---

## 🎯 Цель работы

Изучить жизненный цикл Activity в Android и научиться правильно обрабатывать переходы между состояниями. В ходе работы были освоены:

- **Activity Lifecycle** — последовательность состояний Activity от создания до уничтожения
- **Callback-методы** — onCreate(), onStart(), onResume(), onPause(), onStop(), onDestroy(), onRestart()
- **Логирование** — использование класса Log для отслеживания вызовов методов lifecycle
- **Сохранение состояния** — использование rememberSaveable для сохранения данных при повороте экрана
- **Configuration Changes** — обработка изменений конфигурации устройства (поворот экрана, смена языка)
- **Git и GitHub** — контроль версий и публикация проекта в репозитории

## ❓ Контрольные вопросы

### 1. Какие три метода вызываются при запуске приложения?
При запуске приложения последовательно вызываются три метода:
1. **onCreate()** — Activity создаётся, выполняется инициализация
2. **onStart()** — Activity становится видимой для пользователя
3. **onResume()** — Activity получает фокус и становится активной (можно взаимодействовать)

### 2. Чем отличается onPause() от onStop()?
| Метод | Когда вызывается | Состояние Activity |
|-------|------------------|-------------------|
| **onPause()** | Activity теряет фокус, но остаётся частично видимой | Activity видна, но не в фокусе (например, поверх открыт диалог) |
| **onStop()** | Activity становится полностью невидимой | Activity не видна (пользователь перешёл в другое приложение или нажал Home) |

**Ключевое отличие:** onPause() — Activity ещё видна, onStop() — Activity полностью скрыта.

### 3. Когда вызывается onRestart()?
Метод **onRestart()** вызывается только тогда, когда Activity перезапускается после того, как была остановлена (после вызова onStop()).

**Сценарий:** Пользователь нажал Home → Activity ушла в фон (onPause → onStop) → Пользователь вернулся к приложению из Recent Apps (onRestart → onStart → onResume).

**Важно:** onRestart() НЕ вызывается при обычном запуске приложения, только при возврате из состояния Stop.

### 4. Что происходит при повороте экрана?
При повороте экрана происходит **Configuration Change** (изменение конфигурации):
1. Android полностью **уничтожает** текущую Activity (вызывается onDestroy())
2. Затем **создаёт заново** (вызывается onCreate())
3. **Все данные теряются**, если не использовано сохранение состояния

Это стандартное поведение Android для адаптации UI под новую ориентацию экрана.

### 5. В чём разница между remember и rememberSaveable?
| API | Сохраняет при рекомпозиции | Сохраняет при Configuration Changes |
|-----|---------------------------|-------------------------------------|
| **remember** | ✅ Да | ❌ Нет (данные теряются при повороте) |
| **rememberSaveable** | ✅ Да | ✅ Да (данные сохраняются при повороте) |

**remember** — сохраняет значение только в пределах текущей сессии Activity.  
**rememberSaveable** — сохраняет значение в Bundle, поэтому данные переживают пересоздание Activity.

### 6. Почему onCreate() должен вызывать super.onCreate()?
Вызов **super.onCreate(savedInstanceState)** обязателен, потому что:
- Базовый класс Activity выполняет критически важную инициализацию
- Восстанавливается сохранённое состояние (savedInstanceState)
- Настраивается контекст и другие внутренние компоненты Android
- Без этого вызова приложение может работать некорректно или аварийно завершиться

**Правило:** super.onCreate() должен быть вызван **первым** в методе onCreate().

### 7. Для чего используется класс Log?
Класс **Log** (android.util.Log) используется для:
- **Отладки** — вывод сообщений в консоль Logcat
- **Отслеживания** — мониторинг вызовов методов lifecycle
- **Диагностики** — поиск ошибок и проблем в работе приложения
- **Аудита** — запись информации о действиях пользователя

**Уровни логирования:**
- Log.v() — verbose (подробные сообщения)
- Log.d() — debug (отладочные сообщения)
- Log.i() — info (информационные сообщения)
- Log.w() — warning (предупреждения)
- Log.e() — error (ошибки)

### 8. Какой метод вызывается только один раз за время жизни Activity?
Метод **onCreate()** вызывается только **один раз** за весь жизненный цикл Activity — при её создании.

Все остальные методы (onStart, onResume, onPause, onStop, onRestart) могут вызываться многократно в зависимости от действий пользователя и состояния системы.

## ⚙️ Функциональность

Приложение реализует следующие возможности:

| № | Функция | Описание |
|---|---------|----------|
| 1 | **Кликер баллов** | +5 баллов за каждый клик по изображению |
| 2 | **Система оценок** | 4 уровня: Неудовлетворительно (0-49), Удовлетворительно (50-69), Хорошо (70-89), Отлично (90-100) |
| 3 | **Смена изображений** | 5 разных изображений для разных этапов набора баллов |
| 4 | **Счётчики** | Отображение набранных баллов и количества кликов |
| 5 | **Логирование Lifecycle** | Вывод всех callback-методов в Logcat |
| 6 | **Сохранение состояния** | rememberSaveable сохраняет баллы при повороте экрана |
| 7 | **Светлая тема** | Кастомная цветовая схема с синим акцентом |
| 8 | **Тёмная тема** | Автоматическая адаптация под системную тёмную тему |

### Система оценок:
| Баллы | Оценка | Сообщение |
|-------|--------|-----------|
| 0-49 | Неудовлетворительно | «Продолжайте работать!» |
| 50-69 | Удовлетворительно | «Вы получили зачёт!» |
| 70-89 | Хорошо | «Отличная работа!» |
| 90-100 | Отлично | «Вы успешно сдали курс!» |
